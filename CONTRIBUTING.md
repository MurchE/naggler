# Contributing to Naggler

Thanks for your interest in improving Naggler! Here's how you can help.

## Reporting Bugs

Open a [bug report](https://github.com/MurchE/naggler/issues/new?template=bug_report.md) with:
- Steps to reproduce
- Your OS and Naggler version
- Screenshots if relevant

## Suggesting Features

Open a [feature request](https://github.com/MurchE/naggler/issues/new?template=feature_request.md). Focus on the problem you're solving — the "why" matters more than the "how."

## Improving Documentation

Documentation PRs are welcome. To contribute:

1. Fork the repo and create a branch from `main`
2. Make your changes to README.md, QUICKSTART.md, or other docs
3. Submit a pull request with a clear description of what changed and why

### Style notes for docs
- Keep language direct and concise
- Use second person ("you") not third person ("the user")
- Prefer short sentences over long ones
- No jargon without explanation

## Code Contributions

Naggler's source code is proprietary. If you're interested in contributing code, reach out to naggler@murchewings.com to discuss.

## Code of Conduct

Be respectful and constructive. We're all here to make the tool better.

## Questions?

Email naggler@murchewings.com or open a discussion in [Issues](https://github.com/MurchE/naggler/issues).
